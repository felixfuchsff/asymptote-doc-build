87  // getType() figures out the type of the expression without translating
88  // the code into the virtual machine language or reporting errors to em.
89  // This must follow a few rules to ensure proper translation:
90  // 1. If this returns a valid type, t, trans(e) must return t or
91  // report an error, and transToType(e, t) must run either reporting
92  // an error or reporting no error and yielding the same result as
93  // trans(e).
94  // 2. If this returns a superposition of types (ie. for overloaded
95  // functions), trans must not return a singular type, and every
96  // type in the superposition must run without error properly
97  // if fed to transAsType(e, t).
98  // 3. If this returns ty_error, then so must a call to trans(e) and any

100  // to em.

102  // getType() (or one of the subtypes in case of a superposition)
103  // must report an error.

105  // getType() (or one of the subtypes in case of a superposition)
